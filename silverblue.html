Fedora Silverblue: Save & Switch OS Images

Introduction

Fedora Silverblue is an immutable desktop operating system designed to be reliable and stable. One of the core features of Fedora Silverblue is the ability to easily switch between different OS images (also known as system versions). This guide will show you how to save your current OS image and how to switch to a different image.

1. Saving the Current OS Image

Before making any changes, it’s a good idea to save your current OS image so that you can restore it later if necessary.

  1. Open a Terminal window.
  2. List available OS images: To view the available OS images, run the following command:
    3. ostree admin status

    This will display a list of the current and available OS versions.

  3. Create a backup of the current OS image: To save your current OS image, use the following command:
    4. sudo ostree admin snapshot --create-backup

    This command will create a snapshot of the current OS image. You can restore this snapshot later if needed.

2. Switching Between OS Images

Fedora Silverblue allows you to switch between different OS images using the ostree tool. Here’s how you can switch to a different image:

  1. List available OS images: As mentioned earlier, you can list available OS images with:
    2. ostree admin status

    This will show you the available OS images, including the current one.

  2. Switch to a new OS image: To switch to a different image, use the following command:
    3. sudo ostree admin switch

    Replace with the image you want to switch to (e.g., fedora/33/x86_64/silverblue).

  3. Reboot to apply changes: After switching the image, reboot your system to apply the changes:
    4. sudo systemctl reboot

3. Restoring the Backup Image

If you ever need to restore the OS image you backed up earlier, you can do so by using the following steps:

  1. Restore the backup image: Use this command to restore the snapshot you previously saved:
    2. sudo ostree admin restore-backup

    This will restore the system to the snapshot you created earlier.

  2. Reboot to apply the restored image: Finally, reboot your system:
    3. sudo systemctl reboot

4. Additional Tips

  • Using rollback: You can also roll back to a previous OS image if you don't want to restore a backup manually. Use the command:
    • sudo ostree admin rollback
  • Check for updates: To make sure you are on the latest version of Fedora Silverblue, run:
    • rpm-ostree update

    This will ensure that your system is up to date before switching or rolling back.
